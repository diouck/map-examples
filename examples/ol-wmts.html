Using GeoAdmin WMTS with pure OpenLayers

wmts, openlayers, resolution

Using GeoAdmin WMTS service with only OpenLayers (neither GeoExt nor GeoAdmin).

Sometime, you may only want to use the WMTS service from GeoAdmin with OpenLayers (too complicated, don't trust swisstopo, etc.).

GeoAdmin WMTS service, though compliant with the OGC standard, use some not very common features:

  • It uses it's own projection EPSG:21781. So you can't use it with say, Google Map or OSM
  • The resolutions have to be defined manually, as they do not follow a regular pattern ([650.0, 500.0, 250.0, 100.0, 50.0, 20.0, 10.0, 5.0, 2.5, 2.0, 1.5, 1.0, 0.5]).
  • It's uses a so called Dimension, which are timestamp for each layers.
  • It's use the REST request encoding, unlike most common WMTS services

But beside this, it is easy to use! View the ol-wmts.js source to see how this is done.

Check the official documentation for more details.

Do not forget to add the pro4js library and add the definition for EPSG:21781 (LV03)!

N.B. You have to register to use the WMTS service on your website.
